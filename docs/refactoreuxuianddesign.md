
Each card must have:
- Clear title (small, muted)
- Main content (number / table / list)
- Optional status badge
- Subtle shadow
- Padding is consistent
- No border unless necessary
Status System :
Define a global status color system:
- Success: green
- Warning: yellow/orange
- Danger: red
- Info: blue/gray

Rules:
- Status colors used ONLY for:
  - badges
  - indicators
  - alerts
- Never for decoration
Create KPI cards with:
- Large primary number
- Small label text
- Optional percentage change
- Minimal icon (optional)
- No charts inside KPI cards
Design a list inside a card:
- Simple vertical list
- Primary text bold
- Secondary text muted
- Timestamp aligned right
- Optional status dot
Design a data table inside a card:
- Header background slightly muted
- Zebra rows OR row hover (not both)
- Status shown as badges
- Actions aligned right
- No vertical borders
Each page should have:
- One primary purpose
- One main card or section
- Supporting cards only
- No more than one dominant color
Button rules:
- Primary button only once per page
- Secondary buttons muted
- Destructive actions are red
- Icons only if they improve clarity
Design empty states with:
- Friendly short message
- Neutral icon
- No illustrations
- Clear next action
Design a business SaaS page using the existing dashboard theme.

Constraints:
- Card-driven
- Status-focused
- Same spacing, colors, typography
- Respect visual hierarchy
- No deviation from global design system

Sidebar:
Design a professional SaaS sidebar navigation.

Style:
- Light background
- Subtle separation (shadow or divider)
- No heavy borders
- Clean typography
Sidebar Structure:
Sidebar structure:
1. Logo / App name (top)
2. Main navigation (grouped)
3. Secondary / admin links (bottom)
4. User / settings (very bottom)
Sidebar rules:
- Fixed width (not resizable)
- Icons + text (not icons only)
- Active item clearly highlighted
- Hover state subtle
Sidebar color system:
- Background: neutral (white / light gray)
- Text: dark gray
- Active: muted primary
- Hover: light gray
Navigation Grouping:
Group sidebar items logically:
- Dashboard
- Core Operations
- Reports
- Admin / Settings
Header / Navbar :
Header should contain:
- Page title (left)
- Context actions (filters, date range)
- Global actions (notifications, profile)
Header rules:
- Sticky
- Thin height
- Light background
- Bottom divider OR shadow
Page title rules:
- Clear and explicit
- One line only
- Optional subtitle (small, muted)
Header actions:
- Filters
- Date pickers
- Export
- Primary action (if global)
Button rule:

Only ONE primary in header

Others are outline / ghost
Notifications & User Menu :
Design header utilities:
- Notification bell with badge
- User avatar with dropdown
Rules:
Badge only if unread
Dropdown minimal (profile, settings, logout)
Design a professional SaaS layout with:

- Fixed sidebar navigation
- Sticky top header
- Card-driven content area
- Status-focused UI accents

Constraints:
- Sidebar is clean and minimal
- Header contains page title and actions
- No decorative colors
- Consistent spacing and typography
